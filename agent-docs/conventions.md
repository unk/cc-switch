# 개발 규칙

## 개발 명령

```bash
npm install
npm run build      # tsup → dist/index.js (ESM, shebang 포함, deps 인라인)
npm test           # vitest
npm run lint       # tsc --noEmit + prettier --check
npm run format     # prettier --write
```

버전은 빌드 시 `process.env.CC_SWITCH_VERSION` 으로 주입된다.

## 빠른 검증

```bash
npm run build && npm test && npm run lint
```

- 대화형 수동 확인: `CC_SWITCH_HOME=$(mktemp -d) node dist/index.js create` (TTY 필요).
- 모든 테스트는 `CC_SWITCH_HOME` 으로 격리되어 실제 사용자 환경
  (`~/.zshrc`, `~/.cc-switch`, `~/.local/bin`)을 건드리지 않는다.

## 커밋 규칙 (필수 준수)

- **커밋 메시지에 AI 에이전트 관련 표기를 넣지 않는다.**
  - `Co-Authored-By: Claude`, `Generated by ...`, `Created with AI` 등 금지.
  - 사용자가 **명시적으로 요청한 경우에만** 포함 가능. 기본값은 미포함.
- 커밋/푸시는 사용자가 요청할 때만 수행한다.
